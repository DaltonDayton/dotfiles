# `hyprland` Module Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

> **Status (2026-04-28):** Skeleton landed and the deferred items have all been ported (see `2026-04-26-hyprland-port.md` for chunk-by-chunk tracking + post-port voxtype/NVIDIA hardening). Module ships at `modules/hyprland/` with full app coverage (hypr, waybar, kitty, swaync, wlogout, rofi, matugen, voxtype, sddm-theme/conf, themes, wallpapers). Plan retained as historical record.

**Goal:** Stand up the `modules/hyprland/` mega-module skeleton with seed content from the user's `~/.config/*_new/` work-in-progress directories, retire the standalone `modules/kitty/` module, and validate the framework via `quill list/status/apply`.

**Architecture:** A single quill module under `modules/hyprland/` that owns the Hyprland desktop session — WM configs, bar, terminal, theming engine, wallpapers. Declarative actions (packages, symlinks, files, services) live in `module.toml`; the imperative `install.sh` handles a one-time `matugen` render so a fresh machine has colors after `quill apply`. No new Go code is added — all action types referenced already exist.

**Tech Stack:** Go (existing quill binary), TOML, Bash, Matugen.

**Spec:** `docs/superpowers/specs/2026-04-25-hyprland-module-design.md`

---

## Scope

This plan delivers the **framework** plus the slice of content currently seedable from `~/.config/*_new/` and `~/.config/matugen/`:

**In scope this plan:**
- Remove standalone `modules/kitty/` module; update hosts.
- Create `modules/hyprland/` directory tree and seed: `hypr/`, `waybar/`, `kitty/`, `matugen/`, `wallpapers/` (with one default).
- Author `module.toml` with packages, symlinks, the cliphist file, and the bluetooth service.
- Author `install.sh` with the matugen first-run render.
- Add `hyprland` to both host profiles.
- Smoke test through `quill list` / `quill status` / `quill apply`.

**Deferred to follow-up plans (no usable seed content yet):**
- `swaync` configs (the old configs were generated by the dead theme switcher; need to be written from scratch).
- `voxtype` configs and install.sh setup steps.
- `sddm-theme/`, `sddm.conf`, and the SDDM `[[services]]` entry.
- Device-keyed monitors structure (`hypr_new/monitors.conf` is currently flat — when device variants are added, the `install.sh` device-keyed symlink block from the spec gets enabled and `.gitignore` is added for `monitors.conf` and `voxtype/config.toml`).
- A waybar layout switcher.

The spec describes the full target shape; this plan delivers the skeleton + working bits. Follow-up plans/commits flesh out the deferred items.

## Task 4: Author `install.sh`

**Files:**
- Create: `modules/hyprland/install.sh` (executable)

- [ ] **Step 1: Write `modules/hyprland/install.sh`**

Exact content:

```bash
#!/usr/bin/env bash
set -euo pipefail

MODULE_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"

# --- Matugen first-run render --------------------------------------
# Skip if colors already exist — runtime theme swaps go through a separate
# user-bound script that calls matugen directly. Quill never re-renders.
if [[ ! -f "$HOME/.config/hypr/colors/matugen.conf" ]]; then
  matugen image "$MODULE_DIR/files/wallpapers/default.png"
fi
```
